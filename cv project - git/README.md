# 🔥 Fire & Smoke Vision System

A multi-module computer-vision project for detecting fire and smoke in images.
It goes beyond simple classification — combining **classification, object
detection, segmentation, explainability, and a severity score** into one
desktop application, all trained from a single LabelMe-annotated dataset.

## Modules

| # | Module | What it does | Model |
|---|--------|--------------|-------|
| 1 | **Classification** | Is there fire/smoke? (+ confidence) | ResNet18 (transfer learning) |
| 2 | **Detection** | *Where* is the fire/smoke? (bounding boxes) | YOLOv8 |
| 3 | **Segmentation** | Pixel-level fire/smoke mask | YOLOv8-seg |
| 4 | **Explainability** | *Why* did the model decide "fire"? (Grad-CAM heatmap) | ResNet18 + Grad-CAM |
| 5 | **Severity Score** | How much of the frame is burning? → NONE/LOW/MEDIUM/HIGH/CRITICAL | derived from segmentation |
| 6 | **Evaluation** | Confusion matrix, precision/recall/F1, mAP | scikit-learn / ultralytics |

## How to run

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Train the models (Google Colab, free GPU)
Upload `Fire-Segmentation-Dataset-main.zip` to your Google Drive, then run the
notebooks in `notebooks/` (each one mounts Drive, prepares the dataset, and
trains). Download the resulting weights into the `models/` folder:

- `01_train_classifier.ipynb` → `fire_model.pth`
- `02_train_yolo.ipynb` → `fire_detect.pt` and `fire_seg.pt`

> ⚠️ **Retrain the classifier with `01_train_classifier.ipynb`.** The new
> pipeline uses ImageNet normalization + augmentation and the app's inference
> matches it, so the freshly trained `fire_model.pth` is the one to use.

### 3. (Optional) Prepare the dataset locally
```bash
python src/prepare_dataset.py        # builds dataset/ (classification + yolo_det + yolo_seg)
```

### 4. Run the web app
```bash
python server.py
```
Open **http://localhost:8000**. A React UI (FastAPI backend) — drag in an image,
pick which modules to run, and analyse. No Node/npm needed; the page is served
directly by the backend. Modules whose weights are missing are disabled gracefully.

### 5. Run the end-to-end demo (great report figure)
```bash
python demo.py path/to/image.jpg     # or no argument to use a sample
```
Saves a combined panel (Input · Grad-CAM · Detection · Segmentation) to
`outputs/demo_panel.jpg`.

### 6. Generate evaluation charts for your report
```bash
python src/evaluate.py --data dataset/classification/test --weights models/fire_model.pth
```

## Results

Measured on a held-out test split — see **[RESULTS.md](RESULTS.md)** for the full
breakdown. Headline numbers:

| Module | Metric | Score |
|--------|--------|-------|
| Classification | Test accuracy | **99.0%** |
| Detection | Test mAP50 | **0.787** |
| Detection | Per-class mAP50 | fire 0.754 · smoke 0.802 |
| Segmentation | Test mask mAP50 | **0.792** |
| Segmentation | Per-class mask mAP50 | fire 0.761 · smoke 0.783 |

> `dataset/` is regenerated by `prepare_dataset.py` and can be deleted to save
> space — it's not needed to run the app, only to re-train or re-evaluate.

## Dataset

Fire-Segmentation-Dataset — 5000 images, 3203 with LabelMe polygon annotations
(`fire` = 7802 instances, `smoke` = 3460). A single converter turns these
polygons into every format the project needs, so all modules share one source
of truth.
